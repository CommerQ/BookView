## 1. 背景与目标

### 1.1 背景

用户需要在浏览器中舒适地阅读本地或导入的电子书，并获得接近实体书的翻页与过渡体验。现有纯静态阅读器缺少连贯动效，影响沉浸感。

### 1.2 产品目标

构建 **BookView**：一款基于 Web 的电子书阅读器，支持导入 EPUB、章节导航、阅读进度记忆，并通过 **GSAP** 提供流畅、可访问的翻页与界面过渡动效。

### 1.3 成功标准（可度量）

| 指标 | 目标 | 测量方式 |
|------|------|----------|
| 首屏可交互 | ≤ 2s | `npm run dev` 后打开 `/`；Chrome DevTools Performance，冷启动一次，无扩展 |
| 翻页动画 | 60fps 观感 | 阅读器连续翻页 20 次；Performance 面板无 >50ms 长任务堆积 |
| 无障碍 | 支持 `prefers-reduced-motion` | 系统开启「减少动态效果」后翻页/路由均为瞬时切换 |
| 进度恢复 | 误差 ≤ 1 段 | 阅读到段 N 后刷新或重进 `/read/:id`，显示段 N 或 N±1 |

**测量样例（固定）：** 使用体积约 2–5MB 的 EPUB（含 10+ spine 段）；机器为本地开发机，非 CI。

## 2. 用户与场景

### 2.1 目标用户

- 在桌面或平板浏览器中阅读 EPUB 的个人用户
- 需要快速导入、连续阅读、记住进度的轻度读者

### 2.2 核心场景

| ID | 场景 | 用户行为 | 期望结果 |
|----|------|----------|----------|
| S1 | 首次打开 | 进入应用 | 看到书库空状态与「导入书籍」入口 |
| S2 | 导入书籍 | 选择本地 `.epub` 文件 | 解析成功并出现在书库列表 |
| S3 | 开始阅读 | 点击书籍封面/标题 | 进入阅读视图，显示当前章节内容 |
| S4 | 翻页 | 点击两侧区域 / 键盘方向键 | GSAP 翻页动效（或降级瞬时切换）并加载相邻 spine 段 |
| S5 | 章节跳转 | 打开目录并选择章节 | **瞬时切换**到该 spine 段起始内容（不滚动长文） |
| S6 | 再次阅读 | 重新打开已读书籍 | 自动定位到上次阅读进度 |
| S7 | 删除书籍 | 在书库点击删除 | 书库、IndexedDB、阅读进度一并清除 |

## 3. 功能需求

### 3.1 书库（Library）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| FR-L01 | 支持从本地选择并导入 EPUB 文件 | P0 | 合法 EPUB 导入后出现在书库 |
| FR-L02 | 书库展示书名、作者（若有）、封面缩略图 | P0 | 元数据与 EPUB 一致；封面由 `coverBlob` 生成展示 URL |
| FR-L03 | 支持删除已导入书籍 | P1 | 删除后书库、IndexedDB、该书进度均清除 |
| FR-L04 | 空书库展示引导文案与导入按钮 | P0 | 无书时不可进入阅读页 |

### 3.2 阅读器（Reader）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| FR-R01 | 渲染 EPUB 章节为可阅读 HTML 内容 | P0 | 正文可读，样式不严重错乱 |
| FR-R02 | 按 spine 段切换内容（非全书单页滚动） | P0 | 每次翻页/跳转切换一段 spine |
| FR-R03 | 提供目录面板，列出 spine 章节标题 | P0 | 点击目录项跳转到对应段（同 FR-R02） |
| FR-R04 | 记录每本书的阅读位置（书 ID + 段索引） | P0 | 重开书籍恢复位置 |
| FR-R05 | 支持键盘：← 上一段，→ 下一段，Esc 返回书库 | P1 | 快捷键在 `input/textarea` 无焦点时生效 |

### 3.3 动效（Animation，GSAP）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| FR-A01 | 翻页使用 GSAP 时间轴：旧页退出 + 新页进入 | P0 | 时长 0.35–0.5s，由 `PAGE_TURN_DURATION` 配置 |
| FR-A02 | 书库 ↔ 阅读器路由切换 opacity 过渡 | P1 | 无白屏闪烁；减少动效时 duration 为 0 |
| FR-A03 | `prefers-reduced-motion: reduce` 时跳过动画 | P0 | 翻页与路由均为瞬时切换 |
| FR-A04 | 组件卸载时 `gsap.context` / `matchMedia` 清理 | P0 | 反复进出阅读页无泄漏（控制台无警告） |

### 3.4 存储（Storage）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| FR-S01 | 书籍元数据与 EPUB 二进制存 IndexedDB | P0 | 刷新后书库仍在 |
| FR-S02 | 阅读进度存 localStorage | P0 | 键 `bookview:progress:{bookId}`，与 FR-R04 一致 |

## 4. 非功能需求

| ID | 类别 | 需求 | 验证 |
|----|------|------|------|
| NFR-01 | 性能 | 动效仅 `transform` / `opacity` | Code review + Task 4 单测 |
| NFR-02 | 性能 | 每次只挂载当前 spine 段 HTML | 阅读器 DOM 不包含全书 |
| NFR-03 | 可维护性 | 动效在 `animation/`，路由动效在 `animation/routeFade.ts` | 目录约定 |

| NFR-06 | 可测试性 | storage、epub、pageTurn、LibraryView 有自动化测试 | `npm test` 全绿 |

## 5. 约束与假设

### 5.1 约束

- MVP 仅支持 **EPUB 2/3** 常见结构；不支持 PDF、MOBI
- 首期不实现标注、高亮、全文搜索、云同步
- 动效库固定为 **GSAP** + `@gsap/react`
- 状态管理：**React 组件 state + `storageService`**，不引入 Zustand（MVP）

### 5.2 假设

- 用户通过现代浏览器访问，已启用 JavaScript 与 IndexedDB
- 典型 EPUB 体积 &lt; 30MB；更大文件性能不保证
- 阅读「段」与 EPUB spine item 一一对应

---

## 6. 范围外（Out of Scope）

- 用户账号、多端同步、社交分享
- 字体/theme 商店、TTS 朗读、词典翻译
- 复杂排版（双栏、竖排、注脚浮动布局）
- 付费订阅与 DRM 破解相关能力
